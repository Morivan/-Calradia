# Отчёт по тестированию

## Объём проверки

- 10 сквозных тест-кейсов backend API
- 20 unit-тестов сервисного и сериализационного слоя
- 1 нагрузочный прогон для ключевого endpoint `/api/bootstrap/`
- 1 проверка production-сборки frontend

## Сквозные тест-кейсы

1. Загрузка агрегированного bootstrap payload.
2. Создание отзыва к товару.
3. Полная синхронизация финансового модуля из Google Sheets.
4. Обработка ошибки синхронизации при недоступном Google Sheets.
5. Ответ менеджера demo-клиенту в CRM.
6. Ответ менеджера реальному Telegram-клиенту через Telegram API.
7. Создание заказа из CRM.
8. Ручная публикация маркетингового поста в Telegram.
9. Получение Telegram webhook/polling update и создание клиента в CRM.
10. Автопубликация запланированного маркетингового поста командой backend.

## Unit-покрытие

Покрыты следующие группы логики:

- работа с Google Sheets URL и CSV-данными;
- Telegram service layer;
- логика маркетинговой публикации и расчёта времени публикации;
- сериализация продуктов, материалов, расчётов, клиентов и публикаций.

## Выполненные команды

```powershell
python manage.py test workshop.tests.test_unit workshop.tests.test_integration

```
